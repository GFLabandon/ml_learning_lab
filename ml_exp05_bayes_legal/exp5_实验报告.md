## 1. 实验名称

**法律裁判文书中的案情要素贝叶斯分类**

## 2. 实验目的

1. 理解贝叶斯分类原理，掌握先验概率与后验概率在文本分类中的计算方法。
2. 掌握中文文本预处理完整流程：文本清洗、jieba 分词、停用词过滤与词频统计。
3. 深入理解词袋模型（BoW）与 TF-IDF 两种文本向量化方法的原理及适用场景差异。
4. 实现多项式朴素贝叶斯分类器（MultinomialNB），对法律文书句子进行案情要素多分类预测。
5. 使用 Precision、Recall、F1-Score、混淆矩阵等指标全面评估模型性能，并作出合理分析。

### 4.1 中文文本分类系统框架

一个完整的中文文本分类系统通常包含以下功能模块：

**(1) 文本预处理**：对文档进行分词、去除停用词。中文分词是文本预处理的首要步骤，本实验使用 jieba 分词工具。

**(2) 文本表示**：将文档转化为计算机可处理的数值形式。本实验采用向量空间模型，分别使用词袋模型（BoW）和 TF-IDF 权重方案。

**(3) 文本特征选择**：通过设置 `max_features=300` 保留信息量最大的前 300 个词条，实现维数约简。

**(4) 特征权重计算**：TF-IDF 赋予在特定类别中频繁出现、而在其他类别中少见的词条更高的权重。

**(5) 分类器训练**：使用多项式朴素贝叶斯算法，在训练集上估计各类别的先验概率和条件概率。

**(6) 测试与评价**：使用 Precision、Recall、F1-Score、混淆矩阵等指标评估分类器在测试集上的性能。

### 4.3 TF-IDF 特征权重原理

设词 $w$ 在文档 $d$ 中的 TF-IDF 权重为：

$$\text{TF}(w, d) = \frac{\text{词 } w \text{ 在文档 } d \text{ 中出现的次数}}{\text{文档 } d \text{ 的总词数}}$$

$$\text{IDF}(w) = \log\frac{N + 1}{df(w) + 1} + 1 \quad \text{（sklearn 平滑公式）}$$

$$\text{TF-IDF}(w, d) = \text{TF}(w, d) \times \text{IDF}(w)$$

**核心思想**：某词在某文档中频率高（TF大），但在整个语料中普遍出现（IDF小），则其区分能力弱，权重被压低；若该词仅在少数文档中频繁出现，则 IDF 大，权重高，区分力强。例如"抚养费"在"子女抚养"类中高频出现而在"财产分割"类中少见，其 TF-IDF 权重高，有助于类别区分。

### 4.4 朴素贝叶斯分类原理

**贝叶斯定理**：

$$P(C_k \mid \mathbf{x}) = \frac{P(\mathbf{x} \mid C_k) \cdot P(C_k)}{P(\mathbf{x})}$$

其中：
- $P(C_k)$：类别 $C_k$ 的先验概率，由训练集中各类别频率估计
- $P(\mathbf{x} \mid C_k)$：给定类别 $C_k$ 时文档 $\mathbf{x}$ 出现的似然度
- $P(C_k \mid \mathbf{x})$：后验概率，用于分类决策

**朴素独立假设**（词条之间条件独立）：

$$P(\mathbf{x} \mid C_k) = \prod_{i=1}^{n} P(x_i \mid C_k)$$

**分类决策**（取对数避免下溢）：

$$\hat{C} = \arg\max_{k} \left[ \log P(C_k) + \sum_{i=1}^{n} \log P(x_i \mid C_k) \right]$$

**条件概率估计（拉普拉斯平滑，防止零概率问题）**：

$$P(w \mid C_k) = \frac{N_{w,C_k} + \alpha}{N_{C_k} + \alpha \cdot |V|}$$

其中 $N_{w,C_k}$ 为词 $w$ 在类 $C_k$ 中的出现次数，$N_{C_k}$ 为类 $C_k$ 的总词数，$|V|$ 为词汇表大小，$\alpha=1$（拉普拉斯平滑）。

## 5. 实验过程与源代码

### 5.1 数据加载——核心设计：展平 JSON Lines

本数据集的特殊格式是：文件每一行为一个 JSON **数组**（而非单个对象），因此需要逐行读取、逐行展平：

```python
def load_data_flatten(path):
    all_records = []
    with open(path, 'r', encoding='utf-8') as f:
        for line in f:
            line = line.strip()
            if not line:
                continue
            obj = json.loads(line)
            if isinstance(obj, list):
                all_records.extend(obj)   # 展平：每行是数组
            elif isinstance(obj, dict):
                all_records.append(obj)
    return all_records
```

> **说明**：若使用 `json.load(f)` 整体解析，会在第二行报 `Extra data` 错误。若仅用 `isinstance(item, dict)` 判断单条记录，会因为整行被解析为 `list` 而全部跳过，导致读取 0 条。本实验采用"逐行解析、逐行展平"策略，成功读取 2016 条记录。

### 6.4 结果分析

#### (1) BoW 优于 TF-IDF 的原因分析

在本实验中，BoW 测试准确率（71.35%）略优于 TF-IDF（69.59%）。分析原因：法律文本中存在高度规范化的领域术语，同一类别的关键词（如"婚生子女"、"抚养费"）在该类别中的词频本身已具有强区分力。TF-IDF 的 IDF 因子会压低在多个类别中共同出现的词的权重（如"原告"、"被告"等），但这些词本身在法律文书中携带了重要的语境信息，压低后反而略微损失了信息。

#### (2) 类别不平衡是核心挑战

"子女抚养"占样本总量的 43.8%，是严重的多数类，导致模型对该类别的识别能力极强（F1=0.8690）。而"损害赔偿"（17条）、"过错行为"（25条）、"婚前财产"（8条）等少数类在测试集中样本极少，F1 均为 0，这是典型的**类别不平衡**问题。

#### (3) 混淆矩阵主要误分析

- **"房产处理"误分为"子女抚养"**：离婚诉讼文书中两类句子往往共现，"房产处理"类句子有时也会提及子女或原被告，与"子女抚养"类词汇高度重叠。
- **"分居事实"识别率偏低（Recall=41.67%）**：分居相关描述措辞多样（"离家出走"、"两地分居"、"互不往来"等），词袋模型难以有效泛化。
- **"共同债务"识别准确率高（Precision=85.71%）**：该类句子常含"借款"、"债务"、"偿还"等高辨识度专业术语，特征词权重显著。

#### (4) 过拟合分析

训练准确率（88.34%）远高于测试准确率（71.35%），差值 16.99%，表明模型存在一定过拟合。原因有三：①数据量有限（857条），②类别不平衡导致模型过度拟合多数类，③词袋特征空间维度有限（300维）但涵盖的语言变异不足。

#### (5) 朴素独立假设的影响

法律文本中词语存在强语义依存（如"判决"与"准予"常共现），违反朴素贝叶斯的独立假设。但在高维特征空间中，局部依赖关系在统计上相互抵消，模型仍能取得合理效果，这也是朴素贝叶斯在文本分类中被广泛应用的原因之一。

## 7. 实验结论

1. **数据格式处理是本实验的核心工程挑战**：原始 JSON Lines 文件每行为 JSON 数组，需采用"逐行读取 + 展平"策略，而非直接 `json.load()`，且需对 DV 编码进行中文映射，同时妥善处理空标签，方可完整读取 857 条有效样本。

2. **朴素贝叶斯在法律文本分类中高效实用**：MultinomialNB 训练时间极短（< 0.01s），测试准确率达 71.35%，对多数类（子女抚养、财产分割）分类效果良好，验证了该算法在法律文本分类领域的有效性。

3. **BoW 在本场景中略优于 TF-IDF**：BoW 直接利用词频信息，对高频领域术语的区分力反而优于 TF-IDF，后者的 IDF 因子会压低跨类别通用法律词汇的权重，导致轻微信息损失。

4. **类别不平衡是主要性能瓶颈**：多数类（子女抚养，43.8%）识别 F1 高达 0.869，而少数类（损害赔偿、过错行为等）因测试样本不足，F1 为 0，整体加权 F1（0.688）仍受限于数据分布不均。

5. **改进方向**：
   - **数据层面**：使用 SMOTE 过采样或类别权重（`class_weight`）缓解不平衡问题
   - **特征层面**：引入二元词组（`ngram_range=(1,2)`）捕捉法律文本的词序信息
   - **词典层面**：建立法律专业词典（如"抚养权"、"婚前财产"等复合词）提升分词质量
   - **模型层面**：对比 SVM、逻辑回归；长远可引入 BERT、Legal-BERT 等预训练模型

## 8. 实验心得

本次实验带来了三方面重要收获：

**一、工程调试能力的提升**。数据加载阶段遭遇两个典型障碍：其一，文件每行为 JSON 数组而非 JSON 对象，导致 `isinstance(item, dict)` 过滤掉全部数据（读取0条）；其二，标签为 DV 编码而非中文描述，需要补充映射字典。通过逐步调试、分析数据结构，最终解决了这两个工程问题，锻炼了从错误现象推断根因的能力。

**二、对特征工程重要性的深刻认识**。相同的模型（MultinomialNB）使用 BoW 和 TF-IDF 两种特征方案，测试准确率相差约 1.8 个百分点。这说明在某些场景中，特征表示方法的选择与模型选择同等重要，特征质量对最终性能有决定性影响，不可仅关注算法本身。

**三、对类别不平衡问题的实际体验**。通过混淆矩阵的直观呈现，清晰看到"子女抚养"（44%占比）被过度识别，而"损害赔偿"等少数类完全无法识别。这使我们深刻理解了准确率（Accuracy）作为评价指标的局限性，以及为何需要同时关注 Precision、Recall 和 F1 各类别分布，为处理真实业务场景中的不均衡数据积累了实践经验。
