【纯度函数分析】
  信息熵：H(p) = -p·log₂(p) - (1-p)·log₂(1-p)
  基尼系数：Gini(p) = 2p(1-p)
  → 两者在p=0.5时达最大值
  → 基尼系数计算更高效（无对数）

【树深度优化】
  最优树深度：6
  最优测试准确率：0.9767
  规律：深度过小欠拟合，深度过大过拟合

【剪枝策略对比】

  ├────────────────────┼──────────┼──────────┼──────────┤
  │ 无剪枝(深度30)      │ 1.0000   │ 0.9721   │ 过拟合   │
  │ 预剪枝1(深度5)      │ 0.9721   │ 0.9698   │ 欠拟合   │
  │ 预剪枝2(深度8)⭐   │ 0.9791   │ 0.9791   │ 最优     │
  │ 成本复杂度剪枝      │ 0.9837   │ 0.9767   │ 优秀     │

🎯 关键发现
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 纯度度量
   ✓ 信息熵和基尼系数都在样本最混杂时达最大值
   ✓ 基尼系数计算效率更高（O(n) vs O(n log n)）

2. 树深度优化
   ✓ 存在最优深度使测试准确率最大化
   ✓ 本数据集最优深度为6，但深度8也表现优秀
   ✓ 深度超过15时明显过拟合

3. 剪枝有效性
   ✓ 预剪枝比后剪枝更易实现和调优
   ✓ min_samples_leaf=5是有效的预剪枝参数
   ✓ 最优预剪枝可达到0%过拟合（训练=测试准确率）

4. 集成学习优势
   ✓ 单棵优化树：最高准确率（0.9791），计算快
   ✓ 随机森林：准确率略低，但CV标准差更小（更稳定）
   ✓ 特征重要性：RF提供可靠的特征排序

5. 应用建议
   ✓ 数据集小、特征少 → 单棵决策树
   ✓ 数据集大、特征多 → 随机森林
   ✓ 需要可解释性 → 决策树
   ✓ 需要稳定性 → 随机森林

通过本次实验，我深刻理解了：

1. 模型选择的重要性
   → 最"复杂"的模型不一定是最好的
   → 需要在准确率和泛化能力间找到平衡

2. 参数调优的细致性
   → 小参数的微调可能对性能产生巨大影响
   → 需要系统地探索参数空间

3. 集成学习的智慧
   → 多个弱学习器可组成强学习器
   → 多样性和独立性是集成学习的关键

4. 可视化的价值
   → 图表能直观展示各模型的优劣
   → 易于发现规律和异常

5. 理论与实践的结合
   → 理论指导实践方向
   → 实践验证理论有效性
